Observations​

  • TASKLIST_INDEX_PREFIX is consumed only on 8.7 (Tasklist's own indices, distinct from zeebe-import indices that use ORCH). Also injected via GITHUB_ENV by four playwright_sm_reusable_flow_*.yaml workflows in the cross-component repo and by .github/actions/workflow-vars/action.yaml.
  • OPERATE_INDEX_PREFIX is consumed only on opensearch-embedded.yaml in 8.8/8.9/8.10. Disambiguates CAMUNDA_DATA_SECONDARYSTORAGE_OPENSEARCH_INDEXPREFIX from global.opensearch.prefix (which uses ORCH).
  • ORCHESTRATION_INDEX_PREFIX and OPTIMIZE_INDEX_PREFIX are consumed by every ES/OS persistence layer that overrides defaults. Bundled elasticsearch.yaml / elasticsearch-self-signed.yaml layers rely on chart defaults and reference no prefix vars.

§2 is enforced mechanically by TestUpgradePrefixCoverage.

§3 Maintenance​

When ci-test-config.yaml, persistence YAML, or a cross-component cron nightly changes:

  • Update §1 to reflect the new (or removed) row.
  • §2 is mechanically enforced — diverging on-disk YAML from the matrix fails CI.
